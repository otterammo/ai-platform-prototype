# ADR Index

Architecture Decision Records capture permanent architectural decisions. ADRs
are historical records; do not rewrite old decisions to match later thinking.
When a decision changes, add a new ADR and mark the old one superseded.

Use [template.md](template.md) for new ADRs.

## Statuses

- `Proposed`: The decision is under review.
- `Accepted`: The decision is active.
- `Deprecated`: The decision is still historical but no longer recommended.
- `Superseded`: A newer ADR replaces the decision.
